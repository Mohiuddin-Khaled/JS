var:
  • Function Scope: Variables declared with var are function-scoped. They are only accessible within the function where they are defined.
  • Hoisting: Variables declared with var are hoisted to the top of their containing function or global scope, meaning use can use them before they are declared, but they will be initialized with undefined untill their actual declaration.

    • let:
  • Block Scope: Variables declared with let are block-scoped, meaning they arer accessible only within the block where they are defined. A block can be a function, loop, or any code enclosed in curly braces {}.
  • Hoisting: Variables declared with let are also hoisted to the top of their containing block, but they are not initialized, so you can't use them before their declaration.

    • const:
  • Block Scope: Like let, const, is also block-scoped.
  • Immutability: Variables declared with const are meant to be constants, and their value cannot be reassigned after initialization. However, for object and arrays declared with const, you can still modify their properties or elements, you just can't reassign the variable itself.

    • null:
      Use Cases:
    • Explicit Absence of Value: It can be used to explicitly indicate that a variable, object property, or array element has no value or has been intentionally cleared.
    • As a Placeholder: It can be used as a placeholder value until a meaningful value is assigned.
    • Clearing References: It can be used to release memory or resources by clearing references to objects.
    undefined:
      Use Cases:
    • Uninitialized Variables: Variables that are declared but not initialized are automatically assigned the value undefined.
    • Non-existent Properties: Accessing properties of an object that do not exist returns undefined.
    • Function Returns: Functions that do not return a value implicitly return undefined.

    REST API:

    A REST API (also called a RESTful API or RESTful web API) is an application programming interface (API) that conforms to the design principles of the representational state transfer (REST) architectural style.

      In HTTP 5 methods are used in a REST-based Architecture such as POST, GET, PUT, PATCH, and DELETE:
    • GET: The HTTP GET method is used to read or retrieve a representation of a resource.
    • POST: The POST verb is most often utilized to create new resources. It is used to create subordinate resources.
    • PUT: It can be used to create a resource in the case where the resource ID is chosen by the client instead of by the server.
    • PATCH: The PATCH request only needs to contain the changes to the resource, not the complete resource.
    • DELETE: It is used to delete a resource identified by a URI.
